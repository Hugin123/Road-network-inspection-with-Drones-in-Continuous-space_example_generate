# ALNS + PSO 元启发式算法说明

## 1. 问题描述

### 1.1 场景定义

本算法针对**连续空间断点决策的无人机弧路径问题（UAV Arc Routing Problem with Continuous Breakpoints）**。问题定义如下：

- 路网由一组节点和需求边构成，基站位于节点 0；
- 共有 $K$ 架无人机，每架从基站出发执行巡检任务后返回基站（每架只飞一趟）；
- 所有需求边必须被完整服务（不得遗漏）；
- 每条需求边 $(u, v)$ **可选择**设置一个**断点** $p$，断点位置由连续参数 $\lambda \in (0, 1)$ 决定：
  - 断点将该边分为两段独立子边 $(u \to p)$ 和 $(p \to v)$，两段可由不同无人机、以任意顺序服务；
  - 不设断点时，须由一架无人机从端点 $u$ 到端点 $v$（或 $v$ 到 $u$）连续飞过整条边；
- 无人机在各段子边之间以**欧氏直线**（越野飞行）转移；
- 电池容量约束：每架无人机的**总能量消耗**（巡检 + 转移）不得超过电池上限 $B$。

### 1.2 目标函数

最小化所有无人机的总巡检费用：

$$\min \sum_{k=1}^{K} \left[ c_{\text{call}} \cdot \mathbf{1}[\text{路径}_{k} \neq \emptyset] + \sum_{i} l_{i}^{(k)} \cdot \alpha_{\text{ins}} \cdot c_e + \sum_{j} d_{j}^{(k)} \cdot \alpha_{\text{tr}} \cdot c_e \right]$$

其中：
- $c_{\text{call}}$：每架无人机的调用（启动）成本；
- $l_{i}^{(k)}$：第 $k$ 架无人机巡检第 $i$ 段子边的长度；
- $d_{j}^{(k)}$：第 $k$ 架无人机第 $j$ 次转移飞行距离（含从基站出发和返回基站）；
- $\alpha_{\text{ins}}$：巡检能耗系数（kwh/m）；
- $\alpha_{\text{tr}}$：转移能耗系数（kwh/m）；
- $c_e$：能源单价（元/kwh）。

### 1.3 电池约束

每架无人机的能量消耗须满足：

$$\sum_{i} l_{i}^{(k)} \cdot \alpha_{\text{ins}} + \sum_{j} d_{j}^{(k)} \cdot \alpha_{\text{tr}} \leq B \quad \forall k$$

## 2. 解的表示

### 2.1 数据结构

算法采用三层数据结构表示一个解：

| 类 | 说明 |
|---|---|
| `SubEdge` | 子边，记录所属原始边索引、段类型（0=整边、1=第一段 $u\to p$、2=第二段 $p\to v$）及两端点坐标 |
| `DroneRoute` | 单架无人机路径，包含有序子边列表和每条子边的飞行方向（$a\to b$ 或 $b\to a$） |
| `Solution` | 完整解，包含各无人机路径和每条原始边的断点位置 $\lambda$（`None` 表示无断点） |

### 2.2 断点编码

对第 $i$ 条需求边 $(u_i, v_i)$，断点位置 $\lambda_i \in (0, 1)$ 对应坐标：

$$p_i = u_i + \lambda_i \cdot (v_i - u_i)$$

断点将该边分为：
- 子边 1（seg=1）：从 $u_i$ 到 $p_i$，长度为 $\lambda_i \cdot L_i$；
- 子边 2（seg=2）：从 $p_i$ 到 $v_i$，长度为 $(1-\lambda_i) \cdot L_i$。

## 3. 初始解构建

### 3.1 多启动策略（Multi-Start）

为提升初始解质量，采用多启动策略生成若干候选初始解，取费用最低者：

1. **无断点贪心解**：所有需求边不设断点，直接贪心构建；
2. **全断点贪心解**：所有需求边在中点（$\lambda=0.5$）处设断点，贪心构建；
3. **随机断点贪心解**（重复若干次）：每条边以 50% 概率随机决定是否设断点，断点位置随机取 $[0.15, 0.85]$，贪心构建。

默认运行 10 次，取最优。

### 3.2 贪心路径构建（`greedy_build_solution_from_breakpoints`）

给定断点配置后，按如下贪心规则逐一分配子边：

1. 根据当前断点配置展开所有子边；
2. 对每架无人机，每次选择**当前位置最近的未分配子边**（nearest-neighbor），方向由最小转移距离决定；
3. 若加入某子边后超出电池容量，则跳过，尝试下一架无人机；
4. 若所有无人机均无法容纳某条子边，则强制分配到距离最短的位置（保证可行性）；
5. 最后执行**冗余断点合并**（详见 §3.3）。

### 3.3 冗余断点合并（`merge_redundant_breakpoints`）

若某条有断点的边的两段（seg=1 和 seg=2）被同一架无人机**连续**服务（相邻排列），则该断点为冗余断点——合并两段为整边，将断点设为 `None`，以减少不必要的断点引入，保证解的简洁性。

## 4. ALNS 框架

### 4.1 框架概述

**自适应大邻域搜索（ALNS）** 通过反复执行"破坏—修复"循环探索解空间，并根据历史表现**自适应地调整各算子的使用概率**。

主循环伪代码如下：

```
初始化：multi_start 生成初始解 s₀
        对 s₀ 执行断点邻域搜索
        best ← s₀；current ← s₀

for iteration = 1 to MaxIter do
    选择破坏算子 d（轮盘赌）
    选择修复算子 r（轮盘赌）

    s' ← Repair_r( Destroy_d(current) )

    if iteration % pso_freq == 0 then
        s' ← PSO 优化断点(s')

    执行断点邻域搜索(s')   // 快速模式或完整模式

    if cost(s') < cost(best) then
        best ← s'；score ← σ₁
    elif cost(s') < cost(current) then
        current ← s'；score ← σ₂
    elif SA接受(s') then
        current ← s'；score ← σ₃

    更新算子得分；SA降温

    if iteration % segment_size == 0 then
        归一化更新算子权重

最终：断点邻域搜索 + PSO 精化 best
return best
```

### 4.2 破坏算子

| 算子名称 | 说明 |
|---|---|
| **随机移除**（Random Removal） | 随机选取一定比例的子边从路径中移除 |
| **最差移除**（Worst Removal） | 优先移除"移除收益"最高的子边（即移除该子边后路径费用降低最多的子边） |
| **路径移除**（Route Removal） | 随机选择一架无人机，移除其路径中的一段连续子边 |
| **断点感知移除**（BP-Split Removal） | 优先从有断点的边中移除其中一段子边（造成断点边的"半悬空"状态），迫使修复时产生新的路径结构；若无断点边则退化为随机移除 |

移除比例 $r$ 在每次迭代时从 $[r_{\min}, r_{\max}]$ 中随机抽取。

### 4.3 修复算子

| 算子名称 | 说明 |
|---|---|
| **贪心插入**（Greedy Insert） | 对每条被移除的子边，枚举所有无人机路径的所有插入位置和方向，选择**费用增量最小**且满足电池约束的位置插入 |
| **随机插入**（Random Insert） | 在满足电池约束的可行位置中随机选择一个插入 |

若贪心插入找不到满足约束的位置，则强制插入到距离最短的位置（允许暂时违反电池约束）。

### 4.4 自适应权重更新

每个 **segment**（默认50次迭代）结束后，根据各算子在本 segment 的历史评分更新其选择权重：

$$w_i^{\text{new}} = (1 - \rho) \cdot w_i^{\text{old}} + \rho \cdot \bar{s}_i$$

其中 $\rho$ 为衰减率，$\bar{s}_i$ 为算子 $i$ 在本 segment 的平均得分。权重下限设为 0.1，防止某算子被完全放弃。

评分规则：
- 发现新全局最优 $\sigma_1 = 33$；
- 改进当前解 $\sigma_2 = 9$；
- 被 SA 接受但未改进 $\sigma_3 = 3$；
- 未被接受 = 0。

### 4.5 模拟退火接受准则（SA）

使用模拟退火（Simulated Annealing）作为接受准则，允许以一定概率接受劣解以逃离局部最优：

$$P(\text{接受}) = \exp\!\left(-\frac{\Delta f}{T}\right)$$

其中 $\Delta f = f(s') - f(s_{\text{current}}) > 0$，初始温度 $T_0$ 自动设置为初始解费用的 5%，按冷却系数 $\alpha_T = 0.998$ 逐步降温。

## 5. PSO 断点优化

### 5.1 设计思路

粒子群优化（PSO）负责在 ALNS 搜索到较优路径结构后，**联合优化每条需求边是否设断点以及断点位置**，形成 ALNS 与 PSO 的嵌套协作机制。

### 5.2 决策变量编码

对每条需求边 $i$，定义粒子的第 $i$ 维坐标 $p_i \in [0, 1]$：

$$\lambda_i = \begin{cases} \text{None（不设断点）} & p_i < \theta \\ \dfrac{p_i - \theta}{1 - \theta} & p_i \geq \theta \end{cases}$$

其中 $\theta = 0.4$ 为断点阈值，有效断点范围被限制在 $(0.05, 0.95)$ 以防止断点过于接近端点。

### 5.3 粒子初始化

- 粒子 0：编码当前解的断点配置（保持当前最优信息）；
- 其余粒子：在粒子 0 基础上加随机扰动 $[-0.25, 0.25]$，增强局部探索；
- 最后 $\lceil N/5 \rceil$ 个粒子：完全随机初始化，保证种群多样性。

### 5.4 适应度评估

每个粒子的适应度通过**完整路径重分配**（`greedy_build_solution_from_breakpoints`）评估：解码粒子位置为断点配置，重新贪心构建路径，计算总费用。这与 ALNS 的路径拓扑解耦，允许 PSO 自由探索断点空间。

### 5.5 速度与位置更新

标准 PSO 速度更新：

$$v_i^{(t+1)} = w \cdot v_i^{(t)} + c_1 r_1 (p_{\text{best},i} - x_i^{(t)}) + c_2 r_2 (g_{\text{best},i} - x_i^{(t)})$$

$$x_i^{(t+1)} = \text{clip}(x_i^{(t)} + v_i^{(t+1)},\ 0,\ 1)$$

其中 $w$ 为惯性权重（线性衰减，从初始值按 $\times 0.97$ 衰减，下限 0.35），$c_1 = c_2 = 1.5$ 为学习因子，$r_1, r_2 \sim U(0,1)$。

### 5.6 调用时机

- **主循环中**：每隔 `pso_freq`（默认50）次 ALNS 迭代，对当前解运行一次 PSO；
- **最终精化**：ALNS 主循环结束后，对历史最优解进行 PSO 精化（共3轮：断点邻域搜索 → PSO → 断点邻域搜索）。

## 6. 断点邻域搜索

断点邻域搜索（`_try_add_remove_breakpoints`）是专门针对断点决策的局部搜索，分为两种模式：

### 6.1 快速模式（每次迭代执行）

随机选取少量边（$\leq \lfloor E/3 \rfloor$，$E$ 为边数），对每条选中的边：
- 若无断点 → 尝试在 $\lambda = 0.5$ 或随机位置添加断点；
- 若有断点 → 尝试删除断点。

接受改进则更新当前解。

### 6.2 完整模式（每隔 `pso_freq/2` 次迭代执行）

**阶段1：逐边翻转**

枚举所有需求边，每条边尝试：
- 无断点 → 尝试 7 个候选位置（$0.25, 0.33, 0.5, 0.67, 0.75$ 及 2 个随机值）添加断点；
- 有断点 → 尝试删除，或移动到 7 个候选位置。

**阶段2：两边联合翻转**

随机抽取 $\min(\binom{E}{2},\ 2E)$ 对边，对每对边同时翻转断点状态（有则删、无则加），评估联合改变的效果。

每次改变后均执行完整路径重分配评估，贪心接受改进。

## 7. 算法整体流程图

```
开始
  │
  ▼
多启动策略生成初始解
（无断点贪心 + 全断点贪心 + 多个随机断点贪心，取最优）
  │
  ▼
对初始解执行断点邻域搜索（完整模式）
  │
  ▼
自动设置 SA 初始温度（= 初始费用 × 5%）
  │
  ▼
┌─────────────────────────────────────────────┐
│             ALNS 主循环（MaxIter 次）         │
│                                             │
│  轮盘赌选择破坏算子（4种）                    │
│  轮盘赌选择修复算子（2种）                    │
│         ↓                                   │
│  执行破坏 → 执行修复 → 得到候选解 s'          │
│         ↓                                   │
│  每隔 pso_freq 次：PSO 优化断点              │
│         ↓                                   │
│  断点邻域搜索（快速模式 or 完整模式）          │
│         ↓                                   │
│  SA 接受准则判断是否更新 current             │
│  若优于 best → 更新 best                    │
│         ↓                                   │
│  更新算子得分；SA 降温                        │
│         ↓                                   │
│  每 segment 结束：归一化更新算子权重           │
└─────────────────────────────────────────────┘
  │
  ▼
最终精化：
  断点邻域搜索（完整模式）
  → PSO 优化断点
  → 断点邻域搜索（完整模式）
  │
  ▼
输出最优解（路径、断点、费用、收敛曲线、可视化图）
  │
 结束
```

## 8. 关键参数说明

| 参数 | 默认值 | 说明 |
|---|---|---|
| `max_iter` | 500 | ALNS 最大迭代次数 |
| `segment_size` | 50 | 权重更新周期（每 50 次迭代归一化一次） |
| `removal_min / max` | 0.1 / 0.4 | 破坏算子的子边移除比例范围 |
| `sigma1 / sigma2 / sigma3` | 33 / 9 / 3 | 新全局最优 / 改进当前 / SA接受的奖励分 |
| `decay` | 0.8 | 算子权重更新衰减率 $\rho$ |
| `sa_cooling` | 0.998 | SA 冷却系数 |
| `pso_freq` | 50 | 每隔多少 ALNS 迭代调用一次 PSO |
| `pso_particles` | 15 | PSO 粒子数 |
| `pso_iter` | 20 | PSO 最大迭代次数 |
| `bp_threshold` | 0.4 | PSO 编码中设置断点的阈值 |
| `n_starts` | 10 | 多启动策略的候选初始解数量 |

  服务：边(1,3) (1→bp) [正向]
